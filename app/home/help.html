Undo Help

MS Excel does in general not support "undo" for plug in operations. This applies also for the familiar shortcut ctrl + Z.


The regular Excel Undo function is allows you to undo operations up to the last interaction with a plug in. Once you use a plug in functionality undo is resetted.


PrepJet offers its own "undo" function so that you can go back to a previous state in case the result of a PrepJet function is not what you expected. However font and background colors will not be resetted.

Table Headers

The only condition to use PrepJet is that your table contains headers (e.g. if you have a list with names, the table header might be “name”). Tables shall be unique, i.e. within one sheet, each column has to have a different header.

Lookup Data

Add data from another worksheet to your table using match criteria.


Example: You have a customer list with names, surnames and dates of birth of your customers. You want to add the e-mail addresses from a delivery information table. To assign the correct e-mail address to each customer, you use the name, surname, and date of birth as criteria to match the data.


Select:
1. The target sheet (here: customer list) and the source sheet (here: delivery information table)
2. The column(s) from your source sheet to be added to your target sheet the first table (here: column that contains the e-mail addresses)
3. The first match criteriaon for the data to be added in both tables (here: name)
4. Click “More match criteria” and add the second match criterion (here: surname); repeat for the third match criterion (here:, and birthdates date of birth)

Harmonize Column

Harmonize the character representation of each cell in a column.


Example: You have a column with customer names derived from a registration form. To use the names in marketing emails you want to ensure correct capitalization.


Select:
1. The column(s) in which to harmonize data entries.
2. Select the character representation the data entries should have.

Split Cell Content

Split the data entries in one column and insert them into separate columns.


Example: You have a column that contains different information in the following pattern: Product Name;Product Category;Price;Availability. You want to distribute this information over four columns without harming the data in the columns to the right (which might happen if you use Excel’s Text to columns function).


1. Select the column to be splitted (here: the column with the four attributes)
2. Select the delimiter (i.e. the character that separates your data entries; here: Semicolon)


More Options: In case you only want to split at a certain position of your delimiter, you can specify the position here.


Example: You want the Product Name and the Product Category to be displayed in one column while you want to separate Price and Availability.


Under More Options, select “2” in the Position of Delimiter dropdown, and “From Left” in the Count Direction Dropdown. Now you will get a new column that contains Price;Availability. Now you can repeat the normal split operation with the Semicolon Delimiter.

Extract Text

Extract a part from a column entry based on specified delimiters (delimiter = character that marks the boundaries of the part to be extracted).


Example: You have a column that contains name, surname and date of birth, each separated by a space according to the following pattern: Anthony Miller 12/12/1969. You want to extract the surname and insert it into another column.


1. Select the column from which the part should be extracted (e.g. here the column containing name, surname and date of birth).
2. Specify the start and the end of the part to be extracted (e.g. first whitespace, second whitespace)
3. Select whether the delimiter (e.g. her the whitespace in front of and after the surname) shall be included in your new column.


More Options: Here you can specify the positions of your start and end delimiters. Assuming that some of the persons in your list have more than one first name (e.g. Anthony Stuart Miller), in those cases the second first name would be extracted instead of the surname if you do not specify the delimiter position.


1. Select the count of your start delimiter and the direction from which to count (here: second whitespace from the right, i.e. Dropdown Selections: “2”, “From Right”)


2. Select the count of your end delimiter and the direction from which to count (here: first whitespace from the right, i.e. Dropdown Selections: “1”, “From Right”)

Trim Spaces

Select one or more columns from which to remove all leading and trailing spaces.


Example: you have a column in which some or all entries have leading or trailing spaces (e.g. " John Smith"). Those might cause errors when trying to match cells.


PrepJet’s Trim Spaces function will NOT remove spaces between words in a column (e.g. “ John Smith “ will become “John Smith”).


Simply select one or more columns to remove all leading and trailing spaces.

Custom Inconsistencies

Specify the length of your data entries or certain characters or character sequences the data entries in one column must or must not contain.


Example: you have a "currency" column that indicates the currency in which the price of a product is indicated using abbreviations according to ISO 4217. You want to verify that each data entry consists of exactly three characters.


1. Select the column check (here e.g. the "currency" column).
2. Select "equal to" as operator.
3. Enter the number of characters the entries in your currency column must have (in this case 3) and click apply.


If you leave the text fields for characters your data must or must not contain blank, PrepJet will ignore those fields.

Detect Duplicates

Find and highlight duplicate data records


Example: You have a list of products with the features “category“ and “color“ and you want to find all duplicates based on product name and category.


1. Select the column(s) that shall be included in the duplicate comparison (here e.g. product name and category)
2. Specify the match type: exact vs. fuzzy. “Fuzzy“ means that the data records can differ slightly (e.g. “T-Shirt“ vs. “Tshirt“)


If you select the option “Sort for duplicates”, PrepJet will display all duplicates at the top of your worksheet and place all duplicate pairs below each other. If you do not select this option, PrepJet will only highlight duplicate entries.

Detect Inconsistencies

Find and highlight values that deviate from the data type that occurs most frequently pattern in the column. Data type can be either text, numeric, date or an empty cell. PrepJet assumes the most frequent data type in a column to be the correct one and will highlight all entries that have a different data type. PrepJet will also highlight empty cells. If the majority of cells in a column is empty, PrepJet will ignore the empty cells and screen all populated cells for inconsistent data types.


Example: You have a product catalogue and in some rows, the "size" and "color" columns got confused messed up, so that you have a mix of sizes (numeric data values) and colors (text data values) in one column.


Select the column to detect inconsistencies and apply. If you select the columns and click apply, PrepJet will highlight all numeric entries in the column “colors” and all text entries in the column “size”.

Rule-based Validation

Define rules to validate your data and to detect inconsistent entries based on domain specific conditions. PrepJet will highlight all entries that do not fulfil the condition you defined.


Example: You have a price information table and you know that there are certain price ranges for different product types. You want to find all prices that are not within these ranges. Dresses, for example, are always between 5 and 500 Euro.


In PrepJet, you can define this rule as follows:
If the content in column “Product Type” is equal to (=) “Dress”
then the content in column “Price” has to be between “5” and “500”
If you click the Validate button, PrepJet will highlight all entries in the price column that are smaller than 5 or larger than 500 and where at the same time the product type is “Dress”.
